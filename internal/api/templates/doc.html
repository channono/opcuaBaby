API Documentation

This document provides instructions and examples for interacting with the application's API.

REST API

The REST API provides simple, one-off interactions with OPC UA nodes.

Read Node Attributes

POST /api/v1/read

Reads the attributes of a single OPC UA node.

Request Body (JSON): 
{
    "node_id": "ns=2;s=MyTag1"
}
Example (Python): 
import requests
import json

url = "http://localhost:8080/api/v1/read"
payload = {"node_id": "ns=2;s=MyTag1"}
headers = {'Content-Type': 'application/json'}

response = requests.post(url, data=json.dumps(payload), headers=headers)
print(response.json())

Export All Tags (Address Space)

GET /api/v1/export/tags?format=json|csv

Exports all Variable nodes in the entire address space. Requires active OPC UA connection.

Example (curl, JSON): 
curl -s "http://localhost:8080/api/v1/export/tags?format=json" | jq .
Example (curl, CSV download): 
curl -L "http://localhost:8080/api/v1/export/tags?format=csv" -o tags_all.csv

Export Folder Tags

GET /api/v1/export/tags/folder?node_id=<NodeID>&recursive=true|false&format=json|csv

Exports all Variable nodes under a given folder node. Set recursive to control depth. Requires active OPC UA connection.

Example (curl): 
curl -s "http://localhost:8080/api/v1/export/tags/folder?node_id=ns=2;i=7&recursive=true&format=json" | jq .
Response (JSON item example): 
{
  "node_id": "ns=2;s=MyTag1",
  "name": "MyTag1",
  "data_type": "Int32",
  "description": "",
  "path": ""
}

Write Node Value

POST /api/v1/write

Writes a value to a single OPC UA node.

Request Body (JSON): 
{
    "node_id": "ns=2;s=MyTag1",
    "data_type": "String",
    "value": "Hello, World!"
}
Example (Python): 
import requests
import json

url = "http://localhost:8080/api/v1/write"
payload = {
    "node_id": "ns=2;s=MyTag1",
    "data_type": "String",
    "value": "Hello, World!"
}
headers = {'Content-Type': 'application/json'}

response = requests.post(url, data=json.dumps(payload), headers=headers)
print(response.json())

WebSocket API

The WebSocket API provides real-time updates for subscribed nodes.

Subscribe to Nodes

GET /ws/subscribe

Establishes a WebSocket connection. Once connected, you can send JSON messages to subscribe to or unsubscribe from nodes.

Subscribe Message (JSON): 
{
    "action": "subscribe",
    "node_ids": ["ns=2;s=MyTag1", "ns=2;s=AnotherTag"]
}

On subscribe, the server will:

  • Create server-side watches for those node IDs (if not already).
  • Immediately send one snapshot for each node to this client.
  • After that, push real-time updates on value change.
Unsubscribe Message (JSON): 
{
    "action": "unsubscribe",
    "node_ids": ["ns=2;s=MyTag1"]
}
Subscribe All (JSON): 
{
    "action": "subscribe_all"
}

Note: subscribe_all forwards updates for all server-side watched nodes. To receive updates, either subscribe specific nodes via subscribe, or ensure watches are created by other clients or the server.

Example (JavaScript): 
const socket = new WebSocket("ws://localhost:8080/ws/subscribe");

socket.onopen = function(event) {
    console.log("WebSocket is open now.");
    const msg = {
        action: "subscribe",
        node_ids: ["ns=2;s=MyTag1"]
    };
    socket.send(JSON.stringify(msg));
};

socket.onmessage = function(event) {
    console.log("Received data from server: ", JSON.parse(event.data));
};

socket.onclose = function(event) {
    console.log("WebSocket is closed now.");
};

socket.onerror = function(error) {
    console.error("WebSocket error observed:", error);
};
